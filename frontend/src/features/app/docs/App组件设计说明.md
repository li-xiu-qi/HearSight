# App组件设计说明

## 概述

AppPage组件是HearSight应用的主要容器组件，负责整合整个应用的核心功能模块，包括上传对话框、主工作区布局以及各功能面板的协调管理。

## 组件结构

### AppPage.tsx
应用主容器组件，负责：
- 整体应用布局管理
- 核心功能模块协调
- 全局状态初始化
- 用户会话管理

### 子组件

#### HeaderBar.tsx
应用顶部导航栏，提供：
- 应用Logo展示
- 全局操作按钮（如设置、帮助等）
- 用户信息显示

#### UploadDialog.tsx
文件上传对话框，实现：
- 文件选择和上传
- 上传进度跟踪
- 错误提示和重试机制

#### AppLayout.tsx (新)
工作区布局组件，负责：
- 基于 shadcn/ui resizable 的拖拽面板布局
- 面板尺寸调整和持久化
- 响应式布局适配
- 状态管理和动画效果

## 设计原则

1. **模块化**: 各功能模块独立封装，便于维护和扩展
2. **状态驱动**: 通过状态管理驱动UI变化
3. **用户体验**: 注重操作流畅性和界面友好性
4. **可维护性**: 代码结构清晰，注释完整

## 数据流

- 使用React Context进行全局状态管理
- 组件间通过props传递数据和回调函数
- 异步操作通过自定义hooks处理

## 扩展性

- 遵循组件化设计，易于添加新功能模块
- 通过配置方式支持功能定制
- 预留扩展点便于第三方集成
