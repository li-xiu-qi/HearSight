# UI 组件库设计说明

## 概述

UI 组件库基于 Radix UI 和 Tailwind CSS 构建，提供了一套可复用的基础 UI 组件。这些组件遵循统一的设计语言，确保整个应用的视觉一致性和用户体验。

## 组件列表

### Button (按钮)
提供多种样式和尺寸的按钮组件，支持默认、破坏性、轮廓、次要、幽灵和链接样式。

### Card (卡片)
用于内容分组展示的容器组件，支持标题、描述和内容区域。

### Dialog (对话框)
模态对话框组件，用于重要信息提示或用户确认操作。

### DropdownMenu (下拉菜单)
下拉菜单组件，用于展示一组相关操作选项。

### Form (表单)
表单相关组件集合，包括字段、标签、控件和验证错误提示。

### Input (输入框)
文本输入组件，支持各种输入类型和状态样式。

### ScrollArea (滚动区域)
自定义滚动条容器组件，用于内容超出可视区域时的展示。

### Select (选择器)
下拉选择组件，用于从预定义选项中选择值。

### Separator (分隔符)
视觉分隔线组件，用于内容区域的分割。

### Switch (开关)
布尔值切换组件，用于开启或关闭某项功能。

### Tabs (标签页)
标签页切换组件，用于组织和切换相关内容视图。

## 设计原则

1. **一致性**: 所有组件遵循统一的设计规范和交互模式
2. **可访问性**: 支持键盘导航和屏幕阅读器，符合 WCAG 标准
3. **灵活性**: 组件支持多种变体和自定义属性
4. **性能**: 轻量级实现，最小化重渲染和 DOM 操作

## 使用方式

所有组件均通过 props 进行配置，支持 className 属性进行样式覆盖。组件使用 Tailwind CSS 类名系统，可以方便地与应用的样式体系集成。

## 扩展性

可以通过修改组件变体定义或创建新的变体来扩展组件样式，同时保持与其他组件的一致性。
